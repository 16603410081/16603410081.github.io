模块化
发表于 2022-03-07| 更新于 2022-03-14|javascript
阅读量:

AMD是RequireJs推广过程中对模块定义提出的概念（面对模块加载器）
CMD是SeaJS在模块推广中提出的概念（专注web端很方便的跑在Node环境中）
AMD(提前引入模块)
define（[‘./a’],function（a）{
a.dosomething()
}）

CMD（需要时引入模块）
define(function(require,export,module){
var a=require(‘./a)
a.dosomething()
})
默认是module transport规范（异步规范）
//require([‘./a])数组中可以由外部模块和内部模块
require([‘./a],function(a){
console.log(a)//a需要在函数的型参中再次传入
})
直接写模块名就是commonjs规范（同步规范）
require(‘./a’)

四种模块化方式
1.AMD
2.CMD
3.commonJS
4.ES6 module

定义模块的三种方式
1.define方法可以传递一个参数，但不能是字符串，如果传递一个函数，这个函数内部需要执行commonJs规范
2.define方法可以传递两个参数，第一个参数表示数组则表示注入模块，字符串则表示模块的id，第二个是函数模块，如果第一个的参数传递的是字符串，则如一个参数类似遵守commonJS规范，如传入的数组则遵循module transports规范想要哪个模块就注入哪个模块
3.如果传递三个参数，第一个是id，第二个是数组，第三个是函数模块

如果一个模块没有id可以直接使用，如果没有id则不能直接使用，第一步，模块依赖集中加载模块文件，第二部通过require异步机制指定id导入
define（[‘./test,request’],function(test,request){
car icke=require([‘icke’],function(icke){
console.log(icke)
})
}）

模块覆盖关系：如果定义两个没有id的模块，则前面的覆盖后面的
如果定义两个不同的，则会一require的id为准
如果定义两个相同的，前面的覆盖后面的

暴露接口的六种方式(基于commonJS)
1.module.exports.num
2.exports.num
3.module.exports={
num:
}
如果使用第三种，则第二种会失效
4.module.export=function(){
console.log()
}
5.module.exports=100
当暴露的为类型的时候，则不能添加属性
6.this.num

es modules 暴露对象的方式
define（[],function(){
return 100//通过return暴露接口
commonjs暴露接口的方式可以在esmodules中使用
}）
